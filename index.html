Node Js/Socket.IO

Node JS

  • Nodejs est un serveur d'application qui héberge des applications écrites en javascript.
  • Il s'exécute sur un thread unique et les opération E/S sont non bloquantes
  • Il vient avec npm (gestionnaire de paquets) qui se charge des dépendances

Socket IO

  • Librairie permettant de développer des applications web temps réel
  • Communication bidirectionnelle entre le navigateur client et le serveur
  • Protocole qui s'appuie sur websocket (ou polling si non supportée) de manière transparente.
    • L'avantage: cela permet d'étendre le principe des websockets à plus de navigateurs
    • L'inconvénient: cela restreint le client à communiquer en socketIO

Express

  • "Fast, unopinionated, minimalist web framework for Node.js"

Prérequis

  • Installer nodejs 0.10.25 (iso kermit)
  • Un éditeur javascript(Bracket avec extension extension beautify, IntelliJ Ultimate, Nodeeclipse...)

Init

Récupération du code (tag init)

git clone  <url> -b init
  • Eburo Windows 7: Décompresser le fichiers node_modules.zip
  • Autres: Taper npm install à la racine du projet

Arborescence

  • package.json : descriptif de l'application
  • node_modules: là où sont installées les librairies requises.
  • app.js: l'application express (définission des routes, ressources statiques, etc...)
  • server.js: le serveur socket IO

Exécution

  • Démarrage 
    node server.js
    ou 
    npm start
  • Aller sur: http://localhost:8080/

Routage express

  • Test du routage
  • Changer le titre en passant un paramètre
  • Redémarrage 
    node server.js

Ajout de socket-io

  • Ajout de la dépendance dans package.json
    "socket.io": "~1.3.5"
  • Edition de server.js draw.jade et interactive-draw.js
